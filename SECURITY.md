# Security Policy

## Supported Versions
| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |
| <1.0    | No        |

## Reporting a Vulnerability
Do not open public GitHub issues for sensitive vulnerabilities.

Please report privately with:
- affected version
- reproduction steps
- impact assessment
- suggested remediation (optional)

Contact: `<SECURITY_CONTACT_EMAIL_OR_CHANNEL>`

## Response Targets
- Initial acknowledgement: within 3 business days
- Triage result: within 7 business days
- Fix target: based on severity and release risk

## Scope Notes
Security reports are in scope for:
- authentication/token handling
- upload and package processing flows
- secrets exposure
- code execution and command injection risks
